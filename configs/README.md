## Generating the rest of the grid

Most remaining runs are sweeps over an existing file â€” no new file needed:

```bash
# RQ3 Î±-sweep inside the rationale+answer target:
# alpha=1 weights only answer tokens, but it is not the true answer-only control.
for a in 0.0 0.5 1.0; do
  python -m src.run --config configs/experiments/rq3_qwen2vl_explanation_aware_scienceqa.yaml \
                    --set objective.alpha=$a name=rq3_qwen2vl_alpha${a}_scienceqa
done

# True answer-only uses prompt_variant=answer_only + objective=generative.

# RQ7 transfer is generated by notebooks/00_paper_pipeline.py so source prompt
# family is preserved during target-domain evaluation.
```

For a genuinely new combination (new backbone Ă— dataset), copy the closest file
and change the `model.name` / `data.name` strings.

## Sweeping vs. new files

- **Sweep** (`--set k=v â€¦`) for hyperparameter variants of the same experiment â€”
  Î±, learning rate, max_train. Override `name=` too so outputs don't collide.
- **New file** when the *story* differs (different RQ, backbone, or dataset) and
  you want it version-controlled and self-documenting.
